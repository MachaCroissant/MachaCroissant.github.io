Template Syntax

Vue.js使用了基于HTML的模版语法，在Vue实例的template中直接写入合法的HTML。Vue将模版内容template编译成虚拟DOM渲染函数，结合响应系统智能计算最少需要重新渲染多少组件，最优化DOM操作次数。

Interpolations

Plain Text && Raw HTML

在HTML文件中，将想要传递值的属性用双括号(“Mustache” Syntax)包裹。但是只能在HTML元素两个尖括号之间使用，作为HTML元素内容。

在元素标签中使用v-once可实现只更新一次。

如果想要在双括号中显示经过翻译后的HTML呈现，需要在元素标签中使用v-html，使用之后元素内容会直接全部变成v-html的取值。

Attributes概览

Directives

A directive’s job is to reactively apply side effects to the DOM when the value of its expression changes.

什么是v-属性？它有什么作用？

Directives are prefixed with v- to indicate that they are special attributes provided by Vue, and they apply special reactive behavior to the rendered DOM.

arguments

通过在v-之后跟一个:将值传递给HTML元素的属性。

也可以在:后面跟一个[attributeName/eventname] ，实现“动态参数”，其实这里放入的就是一个JavaScript表达式。

分类

处理元素属性

v-bind:html-attribute="instace-data"让HTML元素的属性与当前JS文件中的传入数据保持一致。html-attribute是directive的”argument“，让v-bind directive将此html-attribute的取值与传入的instace-data相关联。

处理用户输入

v-on:event-listeners="methods-in-our-instance"：将事件监听器绑定到我们对应的方法上。

v-model="app-state-value"：一般用在表单元素中，将表单输入与组件状态协同一致。

条件与循环

v-if="Syntax-that-returns-t/f"：如果赋的值为真，那么该元素就显示。

v-for="list-item in lists"：用于显示实例中的数组中的每一个元素。

Data Properties

实例中的data()是一个函数，每当新建一个实例，Vue就会调用这个函数。data()函数返回一个对象，这个对象可以通过挂在后的app，即const vm = app.mount('#app') 的点访问法vm.$data来访问。访问这个对象属性可以通过两种方法vm.$data.property1或着vm.property1。

当一个Vue实例被创建的时候，它将外部的数据对象的所有property加入到Vue的响应式系统，当属性值发生改变时，视图将会产生“响应”，即匹配更新为新的值。

外部数据对象的属性data.property通过Vue实例点访问vm.property，其中任何一个的变更都会影响到另一个。

当数据被改变时，视图会进行重渲染。只有Vue实例被创建时已经存在于数据对象中的属性才是响应式的。通过使用Object.freeze(data)会阻止修改data中已有的property，意味着响应系统无法再追踪变化。

我们需要保证所有需要用到的数据都在data()函数的return{}中，因为这些实例属性(instance properties)只会在instance第一次被创建的时候被Vue的响应式系统记录在$data对象中。如果存在一些属性你不知道他一开始要取何值，用null undefined占位。

Methods

为了给一个实例(component instance)添加方法是Vue.create({ methods: { }})，在methods内部可以有多个方法。

用this指向当前实例，因为Vue自动将this和当前实例绑定在了一起。注意不要使用箭头函数，因为这会影响Vue的自动绑定执行环境。

为什么这个函数放在computed option中而不是methods option中？

  • 计算属性的值依赖于表达式中的reactive data。

    reactive data的变化会让该计算属性的值自动更新；意味着如果reactive data没有变化，那么计算属性会一直使用上一次计算过的缓存的值。

    如果依赖的值不是reactive data，那么计算属性的值无论如何都不会改变

    Computed properties are cached based on their reactive dependencies.

  • 如果采用方法的形式，就意味着每一次Vue实例的重新渲染都会导致函数调用。

v-for与v-if

当二者存在于同一个元素节点上时，v-if有更高的优先级，总是先测试它。因此v-if没有权限访问v-for作用域内的变量。

如果要实现v-if访问v-for中的变量，可以将v-for先放在template中并将v-if包裹在其中。

Event Handling

事件监听

利用v-on: event-listener="???"(short for @event-listener="???" )。

???处可以接受一个简单的JavaScript语句；也可以接受定义在Vue.createApp的method中的函数名；也可以接受带参数的函数调用(这里也就相当于使用了JavaScript语句)。

当在HTML的内联语句中需要访问原始的DOM事件时，可以将原始DOM事件利用$event特殊变量传入回调函数中。

当对于一个事件存在多个事件处理函数的时候，可以用逗号分开这些函数。

事件修饰符 Event Modifiers

在事件处理函数中经常需要调用 event.preventDefault() 或者 event.stopPropagation()。虽然可以在Vue的methods中去实现，但methods更多需要处理的事数据逻辑而非DOM细节。由此引入事件修饰符，directive.postfix-event-modifiers。

  • .stop
  • .prevent
  • .capture
  • .self
  • .once 点击事件只触发一次，也可以被用到component events中
  • .passive 相当于addEventListener的passive选项，执行事件的默认行为

按键修饰符 Key Modifiers

监听键盘事件时需要检查详细按键，Vue允许为v-on在监听键盘事件时添加按键修饰符。将KeyBoardEvent.key中规定的任意有效按键名转换为kebab-case(短横线形式)作为按键修饰符即可。

表单输入绑定

v-model指令用于表单input textarea 以及 select元素上创建和JavaScript的data对象的双向数据绑定。

本质上是语法糖，负责监听用户输入并更新数据。

v-model会忽略HTML文件中表单元素的各种属性的初始值，只会依据JavaScript文件中的实例数据作为初始数据来源，需要在组件的data选项中声明初始值。

组件

组件在JavaScript文件中就是一个Vue实例，和根实例一样接收相同的如data/computed/watch/methods/lifecycle-hooks等选项，与根实例(root element)的区别在于根实例拥有el的选项。

在HTML文件中被当做可以被重复利用的用户自定义元素(custom element)。

创建自定义组件

全剧注册的组件(根实例的子实例)可以被根实例中的template和根实例中的所有其他子组件使用。

利用props将数据传递给子组件

Props are custom attributes you can register on a component. When a value is passed to a prop attribute, it becomes a property on that component instance.

既然是用户自定义元素，那么应当也有用户自定义元素属性。在定义组件的时候直接引入一个props: []选项即可。然后相对应的在HTML文件中我们就可以使用该元素的标签并在前一个标签中使用props中列出的属性并传递值。
